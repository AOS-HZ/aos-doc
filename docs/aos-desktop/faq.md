# 常见问题

## 总览页没有数据怎么办？

先确认你最近是否运行过 AI 助手任务。如果刚安装 AOS Desktop，可以先正常使用一次 AI 助手，再回到总览页刷新。

如果仍然没有数据，可能是当前工具暂时不在[支持列表](./runtimes.md)里，或相关活动还没有同步到桌面端，等片刻后刷新即可。

## 某个技能为什么没有出现在技能页？

常见原因：

- 技能不在默认扫描路径里。进入**设置 → 通用 → 技能扫描路径**，添加对应目录后刷新。
- 技能刚安装，列表还没有更新。点击技能页的刷新按钮。
- 技能目录结构不完整，暂时无法识别。

## 总览页的 token 数为什么不完整？

部分 AI 工具不支持 token 遥测，总览页会在对应位置标注说明。对于这些工具，你仍然可以查看工具调用次数和安全事件数据。

## 收到告警后要马上删除技能吗？

不一定。告警表示这件事值得关注，不代表一定是恶意行为。

建议先打开告警详情，查看对应会话和技能风险分析。确认来源不可信或行为不符合预期后，再考虑停用或删除。

## "已拦截"是什么意思？

AOS Desktop 的实时保护系统已经阻止了这个操作执行。你仍然应该查看告警详情，确认这次拦截是否符合预期，以及是否需要进一步处理。

## 高风险技能能继续使用吗？

不建议直接继续使用。应该先确认来源、查看可疑行为的具体证据，并让团队成员或负责人复核。确认风险在可接受范围内后，再决定是否继续使用。

## 通知收不到怎么办？

检查以下几点：

1. **设置 → 通知与后台运行**：确认风险通知或会话完成通知已开启。
2. **系统通知权限**：在**设置 → 通知与后台运行**里点击"请求通知权限"，或直接在 macOS 系统设置里检查 AOS Desktop 的通知权限。
3. **通知渠道**：如果收不到某个 Agent 的完成提醒，展开会话完成通知里的**通知渠道**，确认对应 Agent 已开启。
4. **Bark 手机通知**：如果配置了 Bark，确认 Bark 地址已保存，并用**发送测试**验证 iPhone 是否能收到。

更完整的排查步骤见[通知](./notifications.md)。

## 能不能关闭通知？

可以。在**设置 → 通知与后台运行**里，可以单独关闭风险通知、会话完成通知和前台风险卡片。

如果你经常处理重要任务，建议至少保留风险通知。

## 关闭主窗口后应用还在运行吗？

取决于"关闭窗口隐藏到菜单栏"这个设置是否开启：

- **开启**：关闭窗口后应用继续在后台运行，菜单栏图标依然存在。
- **关闭**：关闭窗口时应用会完全退出。

如果你希望 AOS Desktop 持续在后台监控，建议开启这个选项。要完全退出时，点击菜单栏图标，选择**退出 AOS Desktop**。

## 桌面小组件数据不更新怎么办？

小组件每 15 分钟自动刷新一次，需要 AOS Desktop 在后台运行。如果数据长时间不更新，检查菜单栏图标是否存在，或打开主窗口让数据同步后再查看。
