# 支持的 AI 工具

AOS Desktop 通过本机运行时与各类 AI 工具集成，自动采集它们的活动数据。你不需要手动配置，安装并启动 AOS Desktop 后，它会自动识别本机已安装的受支持工具。

## 支持的工具

| 工具                  | 说明                                                     |
| --------------------- | -------------------------------------------------------- |
| **Claude Code**       | Anthropic 官方 CLI，支持 token、工具调用、成本和风险监控 |
| **Codex**             | OpenAI 官方Coding Agent，支持会话和工具调用监控          |
| **Gemini-CLI**        | Google 的 AI 编程助手                                    |
| **Qwen Code**         | 阿里巴巴千问系列 AI 工具                                 |
| **Qoder / QoderWork** | 阿里巴巴 AI 编程工具                                     |

## 每个工具会统计什么

每个已连接的工具都会单独统计以下数据：

- **Token 消耗**：输入 token 和输出 token 分开展示
- **模型调用次数**：调用了多少次语言模型
- **工具调用次数**：执行了多少次外部工具（文件读写、命令执行等）
- **平均工具响应时间**：工具调用的平均耗时
- **成本估算**：根据 token 数和模型定价换算的美元成本
- **缓存节省**：因模型缓存节省的成本（仅部分工具支持）
- **安全事件数**：触发的风险告警数量

在总览页，你可以选择"全部来源"查看汇总数据，也可以单独选择一个工具查看它的专项统计。

## 工具没有出现在列表里

如果某个工具没有出现在来源过滤器或总览页里，可能是：

- 该工具还没有运行过任何任务。
- 该工具当前版本暂不支持数据采集。
- AOS Desktop 还没有完成首次识别，可以等待片刻后刷新。

## 关于 token 统计

部分工具不支持 token 遥测，总览页会标注说明。对于不支持 token 统计的工具，你仍然可以查看工具调用次数和安全事件数据。
