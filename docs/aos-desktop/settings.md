# 设置

设置页分为四个标签：**通用**、**通知与后台运行**、**菜单栏**、**关于**。

## 通用

### 实时保护

开启后，AOS Desktop 会主动拦截检测到的高风险操作。关闭后只记录，不拦截。

如果实时保护显示无法连接，可以点击刷新状态按钮重新检测。

### 关闭行为

- **关闭窗口隐藏到菜单栏**：开启后，点击关闭按钮只会隐藏窗口，应用继续在菜单栏后台运行。关闭后，每次关闭窗口都会完全退出应用。

如果你希望 AOS Desktop 持续在后台监控，建议开启这个选项。

### 数据保留

设置活动数据在本机保留多长时间：

- **7 天**（默认）：适合个人日常使用
- **30 天**：适合需要定期回顾或团队复盘的场景
- **自定义**：自行输入天数

设置页会显示当前数据库占用的存储空间，以及最后一次清理的时间。如果磁盘空间紧张，可以点击**立即清理**手动清除过期数据。

### 技能扫描路径

AOS Desktop 会自动扫描常见目录下的技能。如果你的技能安装在非标准位置，可以在这里手动添加目录，支持多个路径。

添加后，回到技能页点击刷新即可看到新目录下的技能。

## 通知与后台运行

这里可以开启或关闭风险通知、会话完成通知和 Bark 手机通知。完整配置步骤见[通知](./notifications.md)。

### 通知权限

首次使用时，如果系统没有授予通知权限，这里会显示"请求通知权限"按钮。点击后按系统提示操作即可。

### 风险通知

开启后，当检测到新的安全风险时，会通过系统通知提醒你。

适合在以下情况下开启：
- 你经常让 AI 助手执行较长的任务
- 你希望高风险操作第一时间出现在通知栏里
- 你不一定一直盯着 AOS Desktop 主窗口

### 会话完成通知

开启后，当某个 AI 助手任务完成时，会发送通知提醒你回来查看结果。

适合长任务场景：任务跑着的时候你可以去做别的事，完成后通知会提醒你。

### 通知渠道

你可以为"会话完成通知"配置额外的推送渠道，不仅限于系统通知：

| 渠道 | 说明 |
|---|---|
| 系统通知 | macOS 通知中心 |
| Codex / Claude / Gemini / Qwen 等 | 选择哪些 Agent 完成后提醒你 |
| Bark 手机通知 | iPhone 推送服务，需要填写 Bark 测试 URL 或设备 key |

每个渠道可以单独开启/关闭，也可以点击**测试**按钮验证是否配置正确。

如果你要把完成提醒推送到 iPhone，先开启**Bark 手机通知**，再粘贴 Bark app 里的测试 URL 或设备 key。保存后点击**发送测试**，iPhone 收到测试通知就表示配置成功。

### 前台风险卡片

开启后，当应用在前台且有新风险告警时，会在主窗口里弹出一个提醒卡片，显示风险摘要和待处理数量。

如果你觉得卡片出现太频繁，可以关闭这个选项，只保留系统通知。

## 关于

- **应用版本**：当前安装的版本号
- **设备认证状态**：是否已激活
- **运行时引擎版本**：后台引擎的版本
- **打开日志文件夹**：查看应用日志，用于排查问题
- **检查更新**：检测是否有新版本可用
