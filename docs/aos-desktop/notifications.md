# 通知

AOS Desktop 可以在你不盯着主窗口时提醒你：发现新的风险、Agent 完成一次任务，或者把完成提醒同步推送到 iPhone。

![](https://cdn.ytools.xyz/uPic/20260423135700572.jpg)

## 会收到哪些通知

### 风险通知

当 AOS Desktop 发现新的高风险行为时，会通过 macOS 通知提醒你。通知里会显示风险标题和简要原因。

点击通知后，会打开对应的告警详情。你可以在详情里查看这次风险来自哪个 Agent、发生在哪个工作区，以及是否已经被拦截。

风险通知适合一直开启，尤其是你经常让 Agent 在后台执行较长任务时。

### 会话完成通知

当 Agent 完成、失败或取消一次执行后，AOS Desktop 会发送完成提醒。通知通常会包含 Agent 名称、最后的结果摘要和工作区名称，方便你判断是否需要回来处理。

为了避免任务还在继续输出时过早提醒，AOS Desktop 有时会稍等片刻再发送完成通知。如果一次任务没有明确的结束信息或结果摘要，可能不会出现完成提醒。

### Bark 手机通知

如果你希望离开电脑后也能知道任务跑完了，可以开启 Bark 手机通知。开启后，AOS Desktop 在发送会话完成提醒时，会同时通过 Bark 给你的 iPhone 推送一条提醒。

目前 Bark 用于会话完成提醒。风险通知仍然通过 macOS 通知提醒你。

## 开启桌面通知

1. 打开 AOS Desktop。
2. 进入**设置 → 通知与后台运行**。
3. 如果页面顶部出现**请求通知权限**，点击它，并在 macOS 弹窗里允许通知。
4. 开启**风险通知**或**会话完成通知**。

如果你只想接收风险提醒，可以关闭会话完成通知。如果你经常把 Agent 放在后台跑任务，建议两个开关都开启。

## 选择哪些 Agent 会提醒

会话完成通知下面有**通知渠道**按钮。点击后，可以分别开启或关闭不同 Agent 的完成提醒。

只有 AOS Desktop 检测到对应 Agent Runtime 后，这里才会显示它的开关。例如你还没有使用过 Gemini CLI，就可能暂时看不到 Gemini 的通知渠道。

关闭某个渠道后，这个 Agent 的任务完成时不会再提醒你，其他 Agent 不受影响。

## 把完成提醒推送到 iPhone

AOS Desktop 使用 Bark 给 iPhone 发送手机通知。配置前，请先在 iPhone 上安装 Bark，并允许 Bark 发送通知。

1. 在 iPhone 上打开 Bark。
2. 在 Bark 里复制你的测试 URL。找不到位置时，可以打开 [Bark 官方教程](https://bark.day.app/#/tutorial) 查看示意。
3. 回到 Mac，打开 AOS Desktop 的**设置 → 通知与后台运行**。
4. 开启**Bark 手机通知**。
5. 在输入框里粘贴 Bark 测试 URL，或只粘贴设备 key。
6. 点击**保存 Bark 地址**。
7. 点击**发送测试**。

如果 iPhone 收到测试通知，说明配置成功。之后 Agent 完成任务时，你就会在 iPhone 上收到提醒。

## Bark 地址怎么填

你可以粘贴 Bark app 里的完整测试 URL，也可以只粘贴设备 key。

如果 Bark 里显示类似这样的地址：

```text
https://api.day.app/your_key/测试内容
```

可以把整条地址粘贴到 AOS Desktop。AOS Desktop 会自动识别中间的设备 key。

如果你只想填 key，也可以只粘贴：

```text
your_key
```

保存地址后再点击**发送测试**。如果你修改了地址，需要重新保存后才能再次测试。

## 收不到通知怎么办

### Mac 上收不到通知

先检查这些位置：

1. AOS Desktop 的**设置 → 通知与后台运行**里，确认对应通知开关已开启。
2. macOS 的**系统设置 → 通知**里，确认 AOS Desktop 允许发送通知。
3. 检查 Mac 是否打开了专注模式、勿扰模式，或把通知静音了。
4. 确认 AOS Desktop 仍在运行。你可以看菜单栏里是否还有 AOS Desktop 图标。

### 收不到会话完成通知

可以按下面顺序排查：

1. 确认**会话完成通知**总开关已开启。
2. 点击**通知渠道**，确认当前 Agent 的渠道开关已开启。
3. 等任务真正结束后再看通知。有些任务会在停止输出后稍等一会儿才提醒。
4. 如果这次任务没有明确的结束信息或结果摘要，AOS Desktop 可能不会发送完成提醒。

### iPhone 收不到 Bark 通知

先用 AOS Desktop 里的**发送测试**确认 Bark 地址是否可用。

如果测试也收不到：

1. 确认 iPhone 上 Bark 的通知权限已开启。
2. 确认 Bark 地址已经保存，且没有把多余的空格复制进去。
3. 回到 Bark app 重新复制测试 URL，再粘贴到 AOS Desktop。
4. 确认 Mac 和 iPhone 都能正常联网。
5. 如果你使用自建 Bark 服务，确认服务地址可以从当前 Mac 访问。

如果测试能收到，但任务完成时收不到，通常是会话完成通知总开关或对应 Agent 的通知渠道没有开启。
