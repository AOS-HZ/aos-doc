# 活动总览

总览页是 AOS Desktop 的主页，汇总了所有已连接 AI 工具的近期使用情况。它的目的是让你一眼看出最近的用量和风险是否正常，不需要逐条翻看。

![AOS Desktop 活动总览](/images/screenshots/runtime-overview.png)

## 时间范围

总览页顶部可以选择统计时间范围：

- **今天**：当天产生的数据
- **24 小时**：过去 24 小时
- **7 天**：过去一周
- **30 天**：过去一个月
- **全部**：所有历史数据

切换时间范围后，页面上所有指标会同步更新。

## 核心指标

页面顶部的几张卡片显示这段时间内的关键数字：

| 指标 | 说明 |
|---|---|
| Token 消耗 | 所有 AI 工具的 token 使用总量（部分工具如不支持 token 遥测会单独标注） |
| 工具调用次数 | AI 助手执行的外部工具操作总数（文件读写、命令执行等） |
| 平均工具响应时间 | 所有工具调用的平均耗时 |
| 风险事件 | 安全防护系统检测到的风险数量，以及其中已拦截的数量 |
| 模型调用次数 | 调用语言模型的总次数 |
| 成本估算 | 根据 token 数和模型定价换算的美元成本 |
| 缓存节省 | 因模型缓存节省的成本（仅当支持缓存的工具有调用时显示） |

## 工具洞察

工具洞察区域会自动标出值得关注的工具：

- **最慢工具**：平均响应时间最长的工具
- **最繁忙工具**：调用次数最多的工具
- **风险最高工具**：触发告警最多的工具
- **失败率最高工具**：执行失败次数最多的工具

如果这里出现了不熟悉的工具名，或某个工具的失败率突然升高，可以进入会话页查看具体任务。

## Agent 用量分析

Agent 用量分析会展示各个 Agent 的 token 消耗分布，支持图表和表格两种视图。你可以用它对比不同工具或 Agent 的使用占比，找到用量集中的来源。

## 高风险会话

高风险会话列表展示近期风险级别最高的任务记录。点击任意一条可以直接跳转到对应会话，查看风险发生的完整上下文。

## 最近防护事件

最近防护事件展示最新触发的安全告警，包括事件类型和来源。点击可以跳转到告警详情，或直接打开对应任务的时间线。

## 额度快照

额度快照展示当前账户的用量情况，帮助你判断是否接近使用上限。

## 总览页没有数据

如果总览页没有内容，通常是以下原因：

- 还没有运行过 AI 助手任务。
- 当前连接的工具还没有产生数据。
- 数据还在同步中，可以等片刻后刷新。

先正常使用一次 AI 助手，再回来刷新即可看到数据。
